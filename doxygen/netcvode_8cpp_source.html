696 // WATCH statements are rare. Conceptually they are very similar to
697 // PreSyn thresholds as above but an optimal peformance implementation for GPU is
698 // not obvious. Each WATCH statement threshold test could make use of
699 // pscheck. Note that it is possible that there are several active WATCH
700 // statements for a given POINT_PROCESS instance as well as none active.
701 // Also WATCH statements switch between active and inactive state.
702 //
703 // In NEURON,
704 // both PreSyn and WatchCondition were subclasses of ConditionEvent. When
705 // a WatchCondition fired in the fixed step method, it was placed on the queue
706 // with a delivery time of t+teps. WatchCondition::deliver called the NET_RECEIVE
707 // block with proper flag ( but nullptr weight vector). WatchConditions
708 // were created,added/removed,destroyed from a list as necessary.
709 // Perhaps the most commonly used WATCH statement is in the context of a
710 // ThresholdDetect Point_process which watches voltage and compares to
711 // an instance specific threshold parameter. A firing ThresholdDetect instance
712 // would call net_event(tdeliver) which then feeds into the standard
713 // artcell PreSyn sequence (using pntsrc_ instead of thvar_index_).
714 //
715 // So... the PreSyns have the same order as they are checked (although PreSyn
716 // data is AoS instead of SoA and nested 'if' means a failure of SIMD.)
717 // But if multiple WATCH, there is (from one kind of implementation viewpoint),
718 // yet another 'if' with regard to whether a WATCH is active. And if there
719 // are multiple WATCH, the size of the list is dynamic.
720 //
721 // An experimental implementation is to check all WATCH of all instances
722 // of a type with the proviso that there is an active flag for each WATCH.
723 // ie. active, below, var1, var2 are all SoA (except one of the var may
724 // be voltage). Can use 'if (active && pscheck(var1, var2, &below)'
725 // The mod file net_send_buffering fragments can be used which
726 // ultimately call net_send using a transient SelfEvent. ie. all
727 // checking computation takes place in the context of the mod file without
728 // using explicit WatchCondition instances.
